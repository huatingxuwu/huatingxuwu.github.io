## 二、核心机制说明（重要）

### 2.1 文章自动发现

`config.mjs` 内置了 Markdown 文件遍历逻辑（`walkMarkdownFiles` / `collectMarkdownPages`），会自动扫描 `docs/` 下所有 `.md` 文件，提取 frontmatter 中的 `title`、`date`、`description`、`draft` 字段，用于：

- **RSS 生成**（`/rss.xml`）
- **Sitemap 生成**（`/sitemap.xml`）
- **博客列表页**（`/blog`）
- **归档页**（`/archive`）
- **标签页**（`/tags`）

所以新文章只需放在 `docs/` 下，无需额外注册即可出现在博客列表和 RSS 中。

### 2.2 文章 Frontmatter 格式

每篇文章开头应包含 YAML frontmatter：

```yaml
---
title: 文章标题
date: 2026-07-11
description: 一句话描述
draft: false        # true = 草稿，不会出现在博客列表/RSS/Sitemap 中
tags: [密码学, 笔记]  # 可选，用于标签索引
---
```

- `date` 决定文章排序（越新越靠前）和 RSS 发布日期
- `draft: true` 的文章在构建时会被跳过，适合草稿
- `title` 若不写，会自动取文章内第一个 `#` 或 `##` 标题

### 2.3 数学公式

项目已配置 `markdown-it-mathjax3` 插件，支持 LaTeX 数学公式：

- 行内公式：`$E = mc^2$`
- 块级公式：`$$...$$`

无需额外配置。

### 2.4 图片引用（Obsidian 风格）

项目支持 Obsidian 风格的图片引用 `![[图片名.png]]`，图片放在 `docs/extends/` 目录下即可。

语法：
```markdown
![[english_frequency.png]]          # 原始大小
![[english_frequency.png|600]]      # 指定宽度 600px
```

图片会自动从 `docs/extends/` 加载，构建时复制到 `dist/extends/`。

## 三、添加文章到已有专栏

以"现代密码学"专栏为例，假设要添加 `Chapter3.md`。

### 步骤 1：写文章

在 `docs/ModernCryptography/` 下新建 `Chapter3.md`，写好 frontmatter 和正文。

### 步骤 2：更新导航和侧边栏

编辑 `docs/.vitepress/config.mjs`，在 `nav` 和 `sidebar` 中找到"现代密码学"分组，各添加一行：

```js
{ text: '第 3 章笔记', link: '/ModernCryptography/Chapter3' },
```

**注意：nav 和 sidebar 两处都要加**，否则导航下拉菜单和侧边栏会不一致。

### 步骤 3：本地预览

```bash
npm run docs:dev
```

浏览器打开 `http://localhost:5173`，确认文章在导航和侧边栏中可见，内容渲染正常。

## 四、创建全新专栏

假设要新建"数据结构"专栏。

### 步骤 1：新建文件夹和文章

```bash
mkdir docs/DataStructure
# 然后在里面写文章，如 docs/DataStructure/Chapter1.md
```

### 步骤 2：更新 config.mjs

编辑 `docs/.vitepress/config.mjs`。

**① 在 `nav` 数组里加入新专栏下拉菜单：**

```js
{
  text: '数据结构',
  items: [
    { text: '第 1 章', link: '/DataStructure/Chapter1' },
  ],
},
```

**② 在 `sidebar` 数组里加入对应分组：**

```js
{
  text: '数据结构',
  items: [
    { text: '第 1 章', link: '/DataStructure/Chapter1' },
  ],
},
```

### 步骤 3（可选）：更新首页

编辑 `docs/index.md`，在合适位置添加入口，让访客在首页就能看到新专栏。

## 七、关键点速查

| 你要做的事 | 改哪个文件 | 改哪里 |
|-----------|-----------|-------|
| 添加文章到已有专栏 | `config.mjs` | `nav` + `sidebar` 中对应分组的 `items` |
| 新建专栏 | `config.mjs` | `nav` + `sidebar` 各加一个分组 |
| 首页展示新专栏 | `index.md` | 自行添加入口 |
| 改专栏名称 / 排序 | `config.mjs` | 改 `text` 字段或调整数组顺序 |
| 文章设为草稿 | 文章 frontmatter | `draft: true` |
| 添加标签 | 文章 frontmatter | `tags: [标签1, 标签2]` |
| 添加图片 | 放 `docs/extends/` | 文中用 `![[图片名.png]]` 引用 |

## 八、注意事项

1. **文件路径**：`link` 里的路径相对于 `docs/`，不需要 `.md` 后缀。例如 `docs/ModernCryptography/Chapter1.md` → link 写 `/ModernCryptography/Chapter1`。
2. **nav 和 sidebar 同步**：添加文章到已有专栏时，nav 和 sidebar 两处都要更新，否则导航和侧边栏不一致。
3. **frontmatter 必填项**：至少写 `date`，否则文章排序会出问题。
4. **图片**：放在 `docs/extends/` 下，用 `![[文件名.png]]` 引用（Obsidian 兼容语法），不要用 `docs/public/`。
5. **构建产物**：`.vitepress/dist/` 和 `.vitepress/cache/` 已在 `.gitignore` 中，无需手动清理。
