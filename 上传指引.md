# 📖 博客文章上传指引

本指引覆盖添加新文章到 VitePress 博客的全部手动操作步骤。

---

## 一、项目结构速览

```
blog/
├── docs/
│   ├── .vitepress/
│   │   └── config.mjs          ← 导航/侧边栏配置（核心文件）
│   ├── index.md                 ← 首页
│   ├── first-article.md         ← 示例文章
│   ├── ModernCryptography/      ← 专栏示例：现代密码学
│   │   ├── Chapter1.md
│   │   └── Exercise1.md
│   └── 你的新专栏/              ← 新建专栏就新建文件夹
│       └── 文章名.md
├── package.json
└── 上传指引.md                  ← 你正在看的文件
```

---

## 二、添加文章到已有专栏

以"现代密码学"专栏为例，假设要添加 `Chapter2.md`。

### 步骤 1：写文章

在 `docs/ModernCryptography/` 下新建 `Chapter2.md`，写 Markdown 内容即可。

### 步骤 2：更新侧边栏

编辑 `docs/.vitepress/config.mjs`，找到 sidebar 里的"现代密码学"分组，添加新条目：

```js
{
  text: '现代密码学',
  items: [
    { text: '第1章 笔记', link: '/ModernCryptography/Chapter1' },
    { text: '第1章 习题', link: '/ModernCryptography/Exercise1' },
    { text: '第2章 笔记', link: '/ModernCryptography/Chapter2' },  // ← 新增
  ],
},
```

### 步骤 3（可选）：更新导航下拉菜单

如果 nav 里也有这个专栏的下拉菜单，同样在 `config.mjs` 的 `nav` 里加上对应条目：

```js
{
  text: '现代密码学',
  items: [
    { text: '第1章 笔记', link: '/ModernCryptography/Chapter1' },
    { text: '第1章 习题', link: '/ModernCryptography/Exercise1' },
    { text: '第2章 笔记', link: '/ModernCryptography/Chapter2' },  // ← 新增
  ],
},
```

## 三、创建全新专栏

假设要新建一个"数据结构"专栏。

### 步骤 1：新建文件夹和文章

```bash
mkdir docs/DataStructure
# 然后在里面写文章，如 docs/DataStructure/Chapter1.md
```

### 步骤 2：更新 config.mjs

编辑 `docs/.vitepress/config.mjs`。

**① 在 `nav` 数组里加入新专栏**（两种方式）：

方式 A — 直接链接（适合单篇文章的专栏）：
```js
{ text: '数据结构', link: '/DataStructure/Chapter1' },
```

方式 B — 下拉菜单（适合有多篇文章的专栏）：
```js
{
  text: '数据结构',
  items: [
    { text: '第1章', link: '/DataStructure/Chapter1' },
    { text: '第2章', link: '/DataStructure/Chapter2' },
  ],
},
```

**② 在 `sidebar` 数组里加入新分组**：
```js
{
  text: '数据结构',
  items: [
    { text: '第1章', link: '/DataStructure/Chapter1' },
    { text: '第2章', link: '/DataStructure/Chapter2' },
  ],
},
```

### 步骤 3（可选）：更新首页

编辑 `docs/index.md`，在 `features` 或 `hero.actions` 里添加入口，让访客在首页就能看到新专栏。

## 四、本地预览

```bash
npm run docs:dev
```

浏览器打开 `http://localhost:5173`，检查：
- 导航栏是否能看到新专栏
- 侧边栏文章列表是否正确
- 文章内容渲染是否正常（数学公式、图片等）

## 五、构建 & 部署

```bash
# 构建生产版本（输出到 docs/.vitepress/dist）
npm run docs:build

# 提交到 Git
git add .
git commit -m "添加 XXX 文章"
git push
```

部署方式取决于你的设置（GitHub Pages / 其他），按既有流程走即可。

## 六、关键点速查

| 你要做的事 | 改哪个文件 | 改哪里 |
|-----------|-----------|-------|
| 添加文章到已有专栏 | `config.mjs` | `sidebar` 对应分组的 `items` |
| 导航栏也要出现 | `config.mjs` | `nav` 对应下拉菜单的 `items` |
| 新建专栏 | `config.mjs` | `nav` + `sidebar` 各加一个分组 |
| 首页展示新专栏 | `index.md` | `hero.actions` 或 `features` |
| 改专栏名称/排序 | `config.mjs` | 改 `text` 字段或调整数组顺序 |

---

## 七、注意事项

1. **文件路径**：`link` 里的路径是相对于 `docs/` 的，不需要 `.md` 后缀。例如 `docs/ModernCryptography/Chapter1.md` → link 写 `/ModernCryptography/Chapter1`。
2. **子目录**：文章放在 `docs/` 的子文件夹里，路径和 link 保持一致。
3. **数学公式**：如果你的文章用了 LaTeX 数学公式（`$$...$$`），需要确认 VitePress 已配置数学公式插件（如 `markdown-it-mathjax3`），否则公式不会渲染。
4. **图片**：文章里的图片放在 `docs/public/` 下，然后在 Markdown 里用 `/图片名.png` 引用。
