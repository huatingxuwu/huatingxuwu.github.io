### 2.1
#### Question
Prove that, by redefining the key space, we may assume that the key generation algorithm `Gen` chooses a uniform key from the key space, without changing $\Pr[C = c \mid M = m]$ for any $m, c$.

#### Solution
Let $\text{Gen}$ be a randomized key-generation algorithm that uses a sequence of $n$ uniform random bits to output a key.

**1. Redefine the Key Space.** Define a new key space $\mathcal{K}' = \{0, 1\}^n$, where each new "key" $k' \in \mathcal{K}'$ corresponds to a unique random bitstring used by $\text{Gen}$.

**2. Define the New Algorithms.**
- **Gen$'$**: Chooses a key $k'$ uniformly at random from $\mathcal{K}'$. Thus, $\Pr[K' = k'] = 2^{-n}$ for all $k' \in \mathcal{K}'$.
- **Enc$'_{k'}(m)$**: Runs the original encryption algorithm using the key generated by $\text{Gen}$ with the random coins specified by $k'$. That is, $\text{Enc}'_{k'}(m) = \text{Enc}_{\text{Gen}(k')}(m)$.

**3. Equivalence of Ciphertext Distribution.** For any message $m$ and ciphertext $c$:

$$\Pr[C = c \mid M = m] = \sum_{k \in \mathcal{K}} \Pr[K = k] \cdot \Pr[\text{Enc}_k(m) = c]$$

Since $\text{Gen}(k')$ samples $k \in \mathcal{K}$ with its exact original probability distribution when $k'$ is chosen uniformly from $\{0, 1\}^n$, we have:

$$\Pr[C' = c \mid M = m] = \sum_{k' \in \mathcal{K}'} \Pr[K' = k'] \cdot \Pr[\text{Enc}'_{k'}(m) = c]$$

$$\phantom{\Pr[C' = c \mid M = m]} = \sum_{k \in \mathcal{K}} \left( \sum_{k': \text{Gen}(k') = k} \frac{1}{2^n} \right) \cdot \Pr[\text{Enc}_k(m) = c]$$

$$\phantom{\Pr[C' = c \mid M = m]} = \sum_{k \in \mathcal{K}} \Pr[K = k] \cdot \Pr[\text{Enc}_k(m) = c]$$

Therefore, $\Pr[C' = c \mid M = m] = \Pr[C = c \mid M = m]$, maintaining identical security and functionality under a uniform key space $\mathcal{K}'$.

### 2.2
#### Question
Prove that, by redefining the key space as well as the encryption algorithm, we may assume that encryption is deterministic without changing $\Pr[C = c \mid M = m]$ for any $m, c$.

#### Solution
Let $(\text{Gen}, \text{Enc}, \text{Dec})$ be a private-key encryption scheme where the encryption algorithm $\text{Enc}$ is randomized. Suppose $\text{Enc}$ consumes a sequence of $r$ uniform random bits to encrypt a message.

**1. Redefine the Key Space.** Let $\mathcal{K}$ be the original key space. We define a new, expanded key space $\mathcal{K}' = \mathcal{K} \times \{0, 1\}^r$. A new key $k' \in \mathcal{K}'$ is a pair $k' = (k, \rho)$, where $k \in \mathcal{K}$ is the original key and $\rho \in \{0, 1\}^r$ represents the random coins.

**2. Define the New Deterministic Encryption Scheme.**
- **Gen$'$**: Samples $k \leftarrow \text{Gen}$ and samples $\rho \leftarrow \{0, 1\}^r$ uniformly at random. It outputs the combined key $k' = (k, \rho)$.
- **Enc$'_{k'}(m)$**: For $k' = (k, \rho)$, the new encryption algorithm runs the original encryption algorithm deterministically using $\rho$ as its source of randomness:
  $$\text{Enc}'_{(k, \rho)}(m) = \text{Enc}_k(m; \rho)$$
  Because $\rho$ is fixed within the key $k'$, $\text{Enc}'$ is entirely deterministic.

**3. Equivalence of Ciphertext Distribution.** For any message $m$ and ciphertext $c$, the probability of obtaining $c$ in the original randomized scheme is:

$$\Pr[C = c \mid M = m] = \sum_{k \in \mathcal{K}} \Pr[K = k] \cdot \Pr_{\rho \leftarrow \{0, 1\}^r}[\text{Enc}_k(m; \rho) = c]$$

In the modified scheme, since $\text{Gen}'$ chooses $k$ and $\rho$ independently according to their original distributions, the joint probability $\Pr[K' = (k, \rho)] = \Pr[K = k] \cdot 2^{-r}$.

The probability of obtaining $c$ under the new deterministic scheme is:

$$\Pr[C' = c \mid M = m] = \sum_{(k, \rho) \in \mathcal{K}'} \Pr[K' = (k, \rho)] \cdot \mathbb{I}(\text{Enc}'_{(k, \rho)}(m) = c)$$

$$\phantom{\Pr[C' = c \mid M = m]} = \sum_{k \in \mathcal{K}} \sum_{\rho \in \{0, 1\}^r} \Pr[K = k] \cdot 2^{-r} \cdot \mathbb{I}(\text{Enc}_k(m; \rho) = c)$$

$$\phantom{\Pr[C' = c \mid M = m]} = \sum_{k \in \mathcal{K}} \Pr[K = k] \cdot \left( \sum_{\rho \in \{0, 1\}^r} \frac{1}{2^r} \cdot \mathbb{I}(\text{Enc}_k(m; \rho) = c) \right)$$

$$\phantom{\Pr[C' = c \mid M = m]} = \sum_{k \in \mathcal{K}} \Pr[K = k] \cdot \Pr_{\rho \leftarrow \{0, 1\}^r}[\text{Enc}_k(m; \rho) = c]$$

where $\mathbb{I}$ is the indicator function. Thus, $\Pr[C' = c \mid M = m] = \Pr[C = c \mid M = m]$.

Redefining the key space to include the random coins shifts the randomness from the encryption phase to the key-generation phase, making the encryption algorithm itself strictly deterministic.

### 2.3
#### Question
Prove or refute: An encryption scheme with message space $\mathcal{M}$ is perfectly secret if and only if for every probability distribution on $\mathcal{M}$ and every $c_0, c_1 \in \mathcal{C}$ we have $\Pr[C = c_0] = \Pr[C = c_1]$.

#### Solution
**Refute.** The statement is false.

While the condition implies perfect secrecy, the "only if" direction does not hold. Perfect secrecy does not require the ciphertext distribution to be uniform over the ciphertext space.

Here is a counterexample to disprove the statement:

**1. Setup.** Let the message space be $\mathcal{M} = \{0, 1\}$ and the key space be $\mathcal{K} = \{0, 1\}$. Suppose the key generation algorithm chooses $k \in \mathcal{K}$ uniformly at random, so $\Pr[K = 0] = \Pr[K = 1] = 1/2$.

**2. Encryption Scheme.** Define the ciphertext space to be $\mathcal{C} = \{c_0, c_1, c_2\}$. The encryption algorithm $\text{Enc}_k(m)$ is defined as follows:
- $\text{Enc}_k(0) = k$ (maps to $c_0$ if $k=0$, and $c_1$ if $k=1$)
- $\text{Enc}_k(1) = k + 1 \pmod 2$ (maps to $c_1$ if $k=0$, and $c_0$ if $k=1$)

Notice that the ciphertext $c_2 \in \mathcal{C}$ is never output by the encryption algorithm for any message or key. Thus, for any distribution on $\mathcal{M}$, we always have $\Pr[C = c_2] = 0$.

**3. Perfect Secrecy Holds.** By definition, a scheme is perfectly secret if for every $m_0, m_1 \in \mathcal{M}$ and every $c \in \mathcal{C}$:
$$\Pr[C = c \mid M = m_0] = \Pr[C = c \mid M = m_1]$$

- For $c = c_0$: $\Pr[C = c_0 \mid M = 0] = \Pr[K = 0] = 1/2$, and $\Pr[C = c_0 \mid M = 1] = \Pr[K = 1] = 1/2$.
- For $c = c_1$: $\Pr[C = c_1 \mid M = 0] = \Pr[K = 1] = 1/2$, and $\Pr[C = c_1 \mid M = 1] = \Pr[K = 0] = 1/2$.
- For $c = c_2$: $\Pr[C = c_2 \mid M = 0] = 0$, and $\Pr[C = c_2 \mid M = 1] = 0$.

Since the conditional probabilities are equal for all ciphertexts, the scheme is **perfectly secret**.

**4. The Stated Condition Fails.** Consider a distribution on $\mathcal{M}$ where $\Pr[M = 0] = 1$. Under this distribution:

$$\Pr[C = c_0] = \Pr[C = c_0 \mid M = 0]\Pr[M = 0] = \frac{1}{2} \cdot 1 = \frac{1}{2}$$
$$\Pr[C = c_2] = 0$$

Clearly, $\Pr[C = c_0] \neq \Pr[C = c_2]$, which violates the requirement that $\Pr[C = c_0] = \Pr[C = c_1]$ for *every* pair of ciphertexts in $\mathcal{C}$.

### 2.4
#### Question
Prove or refute: For every perfectly secret encryption scheme it holds that for every distribution on the message space $\mathcal{M}$, every $m, m' \in \mathcal{M}$, and every $c \in \mathcal{C}$:
$$\Pr[M = m \mid C = c] = \Pr[M = m' \mid C = c].$$

#### Solution
**Refute.** The statement is false.

The definition of perfect secrecy states that the posterior probability of a message given a ciphertext is equal to its *prior* probability ($\Pr[M = m \mid C = c] = \Pr[M = m]$). It does **not** mean that all messages become equally likely given a ciphertext, unless the prior distribution itself is uniform.

Here is a counterexample to disprove the statement:

**1. Setup.** Let the scheme be a perfectly secret One-Time Pad with $\mathcal{M} = \{0, 1\}$ and $\mathcal{K} = \{0, 1\}$, where $\Pr[K=0] = \Pr[K=1] = 1/2$.

**2. Non-Uniform Message Distribution.** Suppose the prior distribution on $\mathcal{M}$ is **not** uniform. For instance, let:
$$\Pr[M = 0] = \frac{3}{4}, \quad \Pr[M = 1] = \frac{1}{4}.$$

**3. Analysis under Perfect Secrecy.** Since the scheme is perfectly secret, observing any ciphertext $c \in \mathcal{C}$ yields no new information about the message. Therefore:
$$\Pr[M = 0 \mid C = c] = \Pr[M = 0] = \frac{3}{4}$$
$$\Pr[M = 1 \mid C = c] = \Pr[M = 1] = \frac{1}{4}$$

**4. The Stated Condition Fails.** Comparing the two posterior probabilities for the same ciphertext $c$:
$$\Pr[M = 0 \mid C = c] = \frac{3}{4} \neq \frac{1}{4} = \Pr[M = 1 \mid C = c].$$

Since $\Pr[M = m \mid C = c] \neq \Pr[M = m' \mid C = c]$, the statement is refuted.

### 2.5
#### Question
Prove that in Definition 2.6 we may assume $\mathcal{A}$ is deterministic without loss of generality.

#### Solution
**Proof.** In the adversarial indistinguishability experiment $\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}$, a randomized adversary $\mathcal{A}$ chooses two messages $m_0, m_1$, receives a ciphertext $c$, and outputs a bit $b' \in \{0, 1\}$.

Let $r$ be the random coins used by $\mathcal{A}$ from a finite sample space $\mathcal{R}$. The success probability of $\mathcal{A}$ can be expressed by averaging over all possible random coins $r \in \mathcal{R}$:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] = \sum_{r \in \mathcal{R}} \Pr[\mathcal{R} = r] \cdot \Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}(;r), \Pi} = 1]$$

where $\mathcal{A}(;r)$ denotes the deterministic execution of $\mathcal{A}$ when its random coins are fixed to $r$.

By the averaging principle, if a weighted average of a set of values equals a certain probability, there must exist at least one specific choice of random coins $r^* \in \mathcal{R}$ such that the success probability using $r^*$ is greater than or equal to the average success probability:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}(;r^*), \Pi} = 1] \ge \Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]$$

We can define a new, strictly deterministic adversary $\mathcal{A}_{\text{det}}$ that has the optimal random string $r^*$ hardcoded inside it. Whenever $\mathcal{A}_{\text{det}}$ runs, it simply simulates $\mathcal{A}$ using $r^*$.

Since $\mathcal{A}_{\text{det}}$ achieves a success probability at least as high as the original randomized adversary $\mathcal{A}$, any security bound proven against all deterministic adversaries automatically applies to all randomized adversaries. Therefore, assuming $\mathcal{A}$ is deterministic introduces no loss of generality.

### 2.6
#### Question
LEMMA 2.7 Encryption scheme $\Pi$ is perfectly secret if and only if it is perfectly indistinguishable. Prove Lemma 2.7.

#### Solution
**Proof.**

**Definitions:**
- **Perfect Secrecy:** For any distribution over $\mathcal{M}$, any $m \in \mathcal{M}$, and any $c \in \mathcal{C}$ such that $\Pr[C=c] > 0$:
  $$\Pr[M = m \mid C = c] = \Pr[M = m]$$
  An equivalent condition is that for all $m_0, m_1 \in \mathcal{M}$ and all $c \in \mathcal{C}$:
  $$\Pr[\text{Enc}_K(m_0) = c] = \Pr[\text{Enc}_K(m_1) = c]$$
- **Perfect Indistinguishability:** For every adversary $\mathcal{A}$ (without loss of generality, deterministic) competing in the indistinguishability experiment $\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}$, its success probability is exactly $1/2$:
  $$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] = \frac{1}{2}$$

**1. Forward Direction ($\implies$): If $\Pi$ is perfectly secret, then it is perfectly indistinguishable.**

Assume $\Pi$ is perfectly secret. In the experiment $\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi}$, the adversary $\mathcal{A}$ outputs two messages $m_0, m_1 \in \mathcal{M}$. A random bit $b \in \{0, 1\}$ is chosen, and $\mathcal{A}$ receives a challenge ciphertext $c \leftarrow \text{Enc}_K(m_b)$.

The success probability of $\mathcal{A}$ is:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] = \Pr[\mathcal{A}(c) = b]$$

$$\phantom{\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]} = \frac{1}{2}\Pr[\mathcal{A}(c) = 0 \mid b = 0] + \frac{1}{2}\Pr[\mathcal{A}(c) = 1 \mid b = 1]$$

$$\phantom{\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]} = \frac{1}{2}\Pr[\mathcal{A}(\text{Enc}_K(m_0)) = 0] + \frac{1}{2}\left(1 - \Pr[\mathcal{A}(\text{Enc}_K(m_1)) = 0]\right)$$

Because $\Pi$ is perfectly secret, the probability distribution of the ciphertext is completely independent of the underlying message. Thus, for any deterministic rule $\mathcal{A}$ maps to, the probability that $\mathcal{A}$ outputs $0$ is identical whether the message was $m_0$ or $m_1$:

$$\Pr[\mathcal{A}(\text{Enc}_K(m_0)) = 0] = \Pr[\mathcal{A}(\text{Enc}_K(m_1)) = 0]$$

Substituting this back into the success probability formula:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] = \frac{1}{2}\Pr[\mathcal{A}(\text{Enc}_K(m_0)) = 0] + \frac{1}{2} - \frac{1}{2}\Pr[\mathcal{A}(\text{Enc}_K(m_0)) = 0] = \frac{1}{2}$$

Thus, $\Pi$ is perfectly indistinguishable.

**2. Backward Direction ($\impliedby$): If $\Pi$ is perfectly indistinguishable, then it is perfectly secret.**

We prove the contrapositive. Assume $\Pi$ is **not** perfectly secret.

Then there must exist two messages $m_0, m_1 \in \mathcal{M}$ and a ciphertext $c^* \in \mathcal{C}$ such that:

$$\Pr[\text{Enc}_K(m_0) = c^*] \neq \Pr[\text{Enc}_K(m_1) = c^*]$$

Without loss of generality, assume $\Pr[\text{Enc}_K(m_0) = c^*] > \Pr[\text{Enc}_K(m_1) = c^*]$.

We construct a specific deterministic adversary $\mathcal{A}$ to exploit this gap:

1. $\mathcal{A}$ outputs the two distinct messages $m_0$ and $m_1$.
2. Upon receiving the challenge ciphertext $c$, $\mathcal{A}$ checks if $c = c^*$.
   - If $c = c^*$, $\mathcal{A}$ outputs $0$ (guessing $b=0$).
   - If $c \neq c^*$, $\mathcal{A}$ outputs $1$ (guessing $b=1$).

Let's compute the success probability of this adversary $\mathcal{A}$:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] = \frac{1}{2}\Pr[\mathcal{A}(c) = 0 \mid b = 0] + \frac{1}{2}\Pr[\mathcal{A}(c) = 1 \mid b = 1]$$

$$\phantom{\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]} = \frac{1}{2}\Pr[\text{Enc}_K(m_0) = c^*] + \frac{1}{2}\Pr[\text{Enc}_K(m_1) \neq c^*]$$

$$\phantom{\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]} = \frac{1}{2}\Pr[\text{Enc}_K(m_0) = c^*] + \frac{1}{2}\left(1 - \Pr[\text{Enc}_K(m_1) = c^*]\right)$$

$$\phantom{\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1]} = \frac{1}{2} + \frac{1}{2}\left(\Pr[\text{Enc}_K(m_0) = c^*] - \Pr[\text{Enc}_K(m_1) = c^*]\right)$$

Since $\Pr[\text{Enc}_K(m_0) = c^*] - \Pr[\text{Enc}_K(m_1) = c^*] > 0$, we have:

$$\Pr[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1] > \frac{1}{2}$$

This contradicts the assumption of perfect indistinguishability. Thus, if $\Pi$ is perfectly indistinguishable, it must be perfectly secret.

### 2.7
#### Question
What is the ciphertext that results when the plaintext $\text{0x012345}$ (written in hex) is encrypted using the one-time pad with key $\text{0xFFEEDD}$?

#### Solution
In a One-Time Pad (OTP) scheme, encryption is performed using a bitwise XOR ($\oplus$) operation between the plaintext ($M$) and the key ($K$).

Given:
- Plaintext ($M$): $\text{0x012345}$
- Key ($K$): $\text{0xFFEEDD}$

We compute the ciphertext $C = M \oplus K$ byte by byte (or digit by digit in binary):

$$\begin{array}{rccccccc} M: & \text{0x} & 0 & 1 & 2 & 3 & 4 & 5 \\ \oplus\ K: & \text{0x} & \text{F} & \text{F} & \text{E} & \text{E} & \text{D} & \text{D} \\ \hline C: & \text{0x} & \text{F} & \text{E} & \text{C} & \text{D} & \text{9} & \text{8} \end{array}$$

### 2.8
#### Question
For each of the following encryption schemes, state whether the scheme is perfectly secret. Justify your answer in each case.

(a) The message space is $\mathcal{M} = \{0, \dots, 4\}$, and $\text{Gen}$ chooses a uniform key from the key space $\mathcal{K} = \{0, \dots, 5\}$. $\text{Enc}_k(m)$ returns $[m + k \bmod 5]$, and $\text{Dec}_k(c)$ returns $[c - k \bmod 5]$.

(b) The message space is $\mathcal{M} = \{m \in \{0, 1\}^\ell \mid \text{the last bit of } m \text{ is } 0\}$. $\text{Gen}$ chooses a uniform key from $\{0, 1\}^{\ell-1}$. $\text{Enc}_k(m)$ returns ciphertext $m \oplus (k \parallel 0)$, and $\text{Dec}_k(c)$ returns $c \oplus (k \parallel 0)$.

#### Solution
**(a) Not Perfectly Secret (Refuted).** A scheme is perfectly secret if and only if for all $m_0, m_1 \in \mathcal{M}$ and all $c \in \mathcal{C}$, $\Pr[\text{Enc}_K(m_0) = c] = \Pr[\text{Enc}_K(m_1) = c]$.

Here, $|\mathcal{M}| = 5$ (elements $0, 1, 2, 3, 4$) and $|\mathcal{K}| = 6$ (elements $0, 1, 2, 3, 4, 5$). Since the encryption uses $\bmod 5$, the ciphertext space is $\mathcal{C} = \{0, 1, 2, 3, 4\}$. Because $K$ is chosen uniformly at random, $\Pr[K = k] = \frac{1}{6}$ for each $k \in \{0, \dots, 5\}$.

Let's evaluate the probability of getting ciphertext $c = 0$ for two different messages:
- **For $m_0 = 0$:** To get $\text{Enc}_K(0) = [0 + K \bmod 5] = 0$, $K$ can be $0$ or $5$. Thus, $\Pr[\text{Enc}_K(0) = 0] = \Pr[K=0] + \Pr[K=5] = \frac{1}{6} + \frac{1}{6} = \frac{2}{6} = \frac{1}{3}$.
- **For $m_1 = 1$:** To get $\text{Enc}_K(1) = [1 + K \bmod 5] = 0$, $K$ must be $4$ (since $1+4=5 \equiv 0 \bmod 5$). Thus, $\Pr[\text{Enc}_K(1) = 0] = \Pr[K=4] = \frac{1}{6}$.

Since $\Pr[\text{Enc}_K(0) = 0] \neq \Pr[\text{Enc}_K(1) = 0]$, observing the ciphertext leaks statistical information about the plaintext. Therefore, the scheme is **not** perfectly secret.

**(b) Perfectly Secret (Proven).** Let $m \in \mathcal{M}$ be an arbitrary message. By definition, $m$ can be written as $m = m' \parallel 0$, where $m' \in \{0, 1\}^{\ell-1}$ represents the first $\ell-1$ bits.

The encryption algorithm computes:
$$C = m \oplus (K \parallel 0) = (m' \parallel 0) \oplus (K \parallel 0) = (m' \oplus K) \parallel (0 \oplus 0) = (m' \oplus K) \parallel 0$$

Let $c \in \mathcal{C}$ be any valid ciphertext. Since the last bit of any ciphertext generated by this scheme is always $0$, $\mathcal{C}$ consists of all $\ell$-bit strings ending in $0$. Thus, $c$ can be written as $c = c' \parallel 0$, where $c' \in \{0, 1\}^{\ell-1}$.

For any message $m = m' \parallel 0$ and any ciphertext $c = c' \parallel 0$:
$$\Pr[\text{Enc}_K(m) = c] = \Pr[(m' \oplus K) \parallel 0 = c' \parallel 0] = \Pr[m' \oplus K = c'] = \Pr[K = m' \oplus c']$$

Since $K$ is chosen uniformly at random from $\{0, 1\}^{\ell-1}$, the probability that $K$ equals the specific string $m' \oplus c'$ is exactly $\frac{1}{2^{\ell-1}}$.

This probability $\frac{1}{2^{\ell-1}}$ is completely independent of the message $m$. Therefore, for any two messages $m_0, m_1 \in \mathcal{M}$ and any valid ciphertext $c \in \mathcal{C}$, the conditional probabilities are perfectly identical. The scheme is **perfectly secret**.

### 2.9
#### Question
In each of the following schemes, $\text{Enc}_k(m) = [m + k \bmod 3]$. State in each case whether the scheme is perfectly secret, and justify your answers.

(a) The message space is $\mathcal{M} = \{0, 1\}$, and $\text{Gen}$ chooses a uniform key from the key space $\mathcal{K} = \{0, 1\}$.
(b) The message space is $\mathcal{M} = \{0, 1, 2\}$, and $\text{Gen}$ chooses a uniform key from the key space $\mathcal{K} = \{0, 1, 2\}$.
(c) The message space is $\mathcal{M} = \{0, 1\}$, and $\text{Gen}$ chooses a uniform key from the key space $\mathcal{K} = \{0, 1, 2\}$.

#### Solution
Recall that a scheme is perfectly secret if and only if for all $m_0, m_1 \in \mathcal{M}$ and all $c \in \mathcal{C}$:
$$\Pr[\text{Enc}_K(m_0) = c] = \Pr[\text{Enc}_K(m_1) = c]$$

**(a) Not Perfectly Secret (Refuted).** Here, $\mathcal{M} = \{0, 1\}$ and $\mathcal{K} = \{0, 1\}$ uniformly distributed, so $\Pr[K=0] = \Pr[K=1] = 1/2$.

Let's analyze the ciphertext distribution for $c = 2$:
- **For $m_0 = 0$:** To get $\text{Enc}_K(0) = [0 + K \bmod 3] = 2$, we need $K = 2$. Since $2 \notin \mathcal{K}$, this is impossible. Thus, $\Pr[\text{Enc}_K(0) = 2] = 0$.
- **For $m_1 = 1$:** To get $\text{Enc}_K(1) = [1 + K \bmod 3] = 2$, we need $K = 1$. Thus, $\Pr[\text{Enc}_K(1) = 2] = \Pr[K = 1] = 1/2$.

Since $\Pr[\text{Enc}_K(0) = 2] \neq \Pr[\text{Enc}_K(1) = 2]$, the scheme leaks information (if an attacker sees ciphertext 2, they instantly know the message cannot be 0). It is **not** perfectly secret.

**(b) Perfectly Secret (Proven).** Here, $\mathcal{M} = \{0, 1, 2\}$ and $\mathcal{K} = \{0, 1, 2\}$ uniformly distributed, so $\Pr[K=k] = 1/3$ for all $k \in \{0, 1, 2\}$. The ciphertext space is $\mathcal{C} = \{0, 1, 2\}$.

For any message $m \in \mathcal{M}$ and any ciphertext $c \in \mathcal{C}$, the equation $c \equiv m + k \pmod 3$ has exactly one unique solution for $k$:
$$k \equiv c - m \pmod 3$$

Since $c - m \bmod 3$ always yields a valid integer in $\{0, 1, 2\}$, there is exactly one key that maps any $m$ to any $c$. Thus, for every $m \in \mathcal{M}$ and $c \in \mathcal{C}$:
$$\Pr[\text{Enc}_K(m) = c] = \Pr[K = (c - m \bmod 3)] = \frac{1}{3}$$

Because this probability is $1/3$ for all messages, the ciphertext distribution is completely independent of the plaintext. The scheme is **perfectly secret**. (This is the shift cipher version of the One-Time Pad over $\mathbb{Z}_3$.)

**(c) Perfectly Secret (Proven).** Here, $\mathcal{M} = \{0, 1\}$ and $\mathcal{K} = \{0, 1, 2\}$ uniformly distributed, so $\Pr[K=k] = 1/3$ for all $k \in \{0, 1, 2\}$. The ciphertext space is $\mathcal{C} = \{0, 1, 2\}$.

Let's evaluate $\Pr[\text{Enc}_K(m) = c]$ for every combination:
- **For $m = 0$:**
  - $\text{Enc}_K(0) = 0 \implies K = 0 \implies \Pr = 1/3$
  - $\text{Enc}_K(0) = 1 \implies K = 1 \implies \Pr = 1/3$
  - $\text{Enc}_K(0) = 2 \implies K = 2 \implies \Pr = 1/3$
- **For $m = 1$:**
  - $\text{Enc}_K(1) = 0 \implies K = 2 \implies \Pr = 1/3$
  - $\text{Enc}_K(1) = 1 \implies K = 0 \implies \Pr = 1/3$
  - $\text{Enc}_K(1) = 2 \implies K = 1 \implies \Pr = 1/3$

For both possible messages $m \in \{0, 1\}$ and any ciphertext $c \in \{0, 1, 2\}$, we have $\Pr[\text{Enc}_K(m) = c] = 1/3$. Since the conditional probabilities are perfectly identical across the message space, the scheme is **perfectly secret**.

### 2.10
#### Question
The following questions concern the message space $\mathcal{M} = \{0, 1\}^{\le \ell}$, the set of all nonempty binary strings of length at most $\ell$.

(a) Consider the encryption scheme in which $\text{Gen}$ chooses a uniform key from $\mathcal{K} = \{0, 1\}^\ell$ and $\text{Enc}_k(m)$ outputs $k_{|m|} \oplus m$, where $k_t$ denotes the first $t$ bits of $k$. Show that this scheme is not perfectly secret for message space $\mathcal{M}$.

(b) Design a perfectly secret encryption scheme for message space $\mathcal{M}$.

#### Solution
**(a) Proof that the scheme is not perfectly secret.** A scheme is perfectly secret if and only if the ciphertext distribution is completely independent of the message plaintext. However, this scheme leaks the exact length of the plaintext, because the ciphertext $c = k_{|m|} \oplus m$ has the same bit-length as the message ($|c| = |m|$).

We can formally refute perfect secrecy using two distinct messages of different lengths:

1. Consider $m_0 = 0$ (length 1) and $m_1 = 00$ (length 2), both belonging to $\mathcal{M}$ (assuming $\ell \ge 2$).
2. Let $c^* = 1$ be a chosen ciphertext of length 1.
3. Calculate the conditional probabilities:
   - **For $m_0$:** $\Pr[\text{Enc}_K(m_0) = c^*] = \Pr[K_1 \oplus 0 = 1] = \Pr[K_1 = 1] = \frac{1}{2}$
   - **For $m_1$:** Since $\text{Enc}_K(m_1)$ always produces a ciphertext of length $|m_1| = 2$, it can never yield the 1-bit ciphertext $c^*$. Thus, $\Pr[\text{Enc}_K(m_1) = c^*] = 0$.

Since $\Pr[\text{Enc}_K(m_0) = c^*] \neq \Pr[\text{Enc}_K(m_1) = c^*]$, an observer gains critical information about the message length from the ciphertext. Therefore, the scheme is **not** perfectly secret.

**(b) Design of a perfectly secret encryption scheme.** To achieve perfect secrecy for variable-length messages up to length $\ell$, the scheme must hide both the content and the original length of the message. We can achieve this by padding all messages to a fixed maximum length $\ell$ before applying a standard One-Time Pad.

**Construction:**
- **Gen**: Chooses a uniform key $k$ from the expanded key space $\mathcal{K}' = \{0, 1\}^{\ell + \lfloor \log_2 \ell \rfloor + 1}$ (or simply a key of sufficient length to cover the maximum padded string length).
- **Enc$_k(m)$**:
  1. **Padding Phase:** Convert $m$ into a fixed-length string of size $L$ using an invertible parsing mechanism. For example, append a `1` bit followed by enough `0` bits to fill the rest:
     $$\text{pad}(m) = m \parallel 1 \parallel 0^{\ell - |m|}$$
     The padded string $\text{pad}(m)$ always has a fixed length of exactly $\ell + 1$ bits, regardless of how short $m$ was originally.
  2. **XOR Phase:** Generate a uniform random key $k$ of length $\ell + 1$. Compute the ciphertext:
     $$c = \text{pad}(m) \oplus k$$
- **Dec$_k(c)$**:
  1. Compute the padded block: $x = c \oplus k$.
  2. Locate the trailing `1` bit from the right side of $x$, strip it and all trailing `0`s following it away to recover the original message $m$.

### 2.11
#### Question
When using the one-time pad with the key $k = 0^\ell$, we have $\text{Enc}_k(m) = k \oplus m = m$ and the message is sent in the clear! It has therefore been suggested to modify the one-time pad by only encrypting with $k \neq 0^\ell$ (i.e., to have $\text{Gen}$ choose $k$ uniformly from the set of *nonzero* keys of length $\ell$). Is this modified scheme still perfectly secret? Explain.

#### Solution
**No**, the modified scheme is **not** perfectly secret.

**Explanation.** A scheme is perfectly secret if and only if for all $m_0, m_1 \in \mathcal{M}$ and all $c \in \mathcal{C}$, we have $\Pr[\text{Enc}_K(m_0) = c] = \Pr[\text{Enc}_K(m_1) = c]$.

By removing the zero key $0^\ell$, the key space becomes $\mathcal{K}' = \{0, 1\}^\ell \setminus \{0^\ell\}$, with size $|\mathcal{K}'| = 2^\ell - 1$. Each key is chosen uniformly at random, so $\Pr[K = k] = \frac{1}{2^\ell - 1}$ for all $k \in \mathcal{K}'$.

Let's test this with a counterexample by setting the ciphertext $c$ to be identical to one of our test messages:

1. Choose two distinct messages $m_0$ and $m_1$ from $\mathcal{M} = \{0, 1\}^\ell$.
2. Let the target ciphertext be $c^* = m_0$.
3. Compute the conditional probabilities:
   - **For $m_0$:** To get $\text{Enc}_K(m_0) = m_0$, we need the key to be $K = m_0 \oplus m_0 = 0^\ell$. However, the zero key has been removed ($0^\ell \notin \mathcal{K}'$). Therefore, this event is impossible:
     $$\Pr[\text{Enc}_K(m_0) = m_0] = 0$$
   - **For $m_1$:** To get $\text{Enc}_K(m_1) = m_0$, we need the key to be $K = m_1 \oplus m_0$. Since $m_0 \neq m_1$, this key is nonzero ($m_1 \oplus m_0 \neq 0^\ell$), meaning it belongs to our valid key space $\mathcal{K}'$. Therefore:
     $$\Pr[\text{Enc}_K(m_1) = m_0] = \Pr[K = m_1 \oplus m_0] = \frac{1}{2^\ell - 1}$$

Since $\Pr[\text{Enc}_K(m_0) = m_0] \neq \Pr[\text{Enc}_K(m_1) = m_0]$, the ciphertext leaks statistical information. Specifically, if an attacker intercepts a ciphertext $c$, they instantly learn that the plaintext $m$ **cannot** be equal to $c$. Thus, perfect secrecy is destroyed.

### 2.12
#### Question
Let $\Pi$ denote the Vigenère cipher where the message space consists of all 3-character strings (over the English alphabet), and the period $t$ is fixed to 2 (and so the key is a uniform string of length 2). Define $\mathcal{A}$ as follows: $\mathcal{A}$ outputs $m_0 = \text{aaa}$ and $m_1 = \text{aab}$. When given a ciphertext $c$, it outputs 0 if the first character of $c$ is the same as the third character of $c$, and outputs 1 otherwise. Compute $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right]$.

#### Solution
In the Vigenère cipher with period $t=2$, a key is a pair $K = (k_1, k_2)$ chosen uniformly at random from $\{0, \dots, 25\}^2$. For a 3-character message $m = m[1]m[2]m[3]$, the ciphertext $c = c[1]c[2]c[3]$ is computed character-by-character using shift addition modulo 26:

$$c[1] = (m[1] + k_1) \bmod 26$$
$$c[2] = (m[2] + k_2) \bmod 26$$
$$c[3] = (m[3] + k_1) \bmod 26$$

Notice that the first and third characters of the message are encrypted using the same key component $k_1$.

The adversary's success probability in the experiment is:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] = \frac{1}{2}\Pr[\mathcal{A}(c) = 0 \mid b = 0] + \frac{1}{2}\Pr[\mathcal{A}(c) = 1 \mid b = 1]$$

Let's evaluate the behavior of $\mathcal{A}$ under both cases:

**1. Case $b = 0$ (Message is $m_0 = \text{aaa}$).** Here, $m_0[1] = \text{'a'} = 0$ and $m_0[3] = \text{'a'} = 0$. The ciphertext characters are:
$$c[1] = (0 + k_1) \bmod 26 = k_1$$
$$c[3] = (0 + k_1) \bmod 26 = k_1$$
Since $c[1] = c[3]$ always holds regardless of the choice of $k_1$, the condition "$c[1] == c[3]$" is always true. Therefore, $\mathcal{A}$ always outputs $0$:
$$\Pr[\mathcal{A}(c) = 0 \mid b = 0] = 1$$

**2. Case $b = 1$ (Message is $m_1 = \text{aab}$).** Here, $m_1[1] = \text{'a'} = 0$ and $m_1[3] = \text{'b'} = 1$. The ciphertext characters are:
$$c[1] = (0 + k_1) \bmod 26 = k_1$$
$$c[3] = (1 + k_1) \bmod 26 = k_1 + 1 \bmod 26$$
Since $c[3] = c[1] + 1 \bmod 26$, it is mathematically impossible for $c[1]$ to equal $c[3]$. Thus, the condition "$c[1] == c[3]$" is always false. Therefore, $\mathcal{A}$ always outputs $1$:
$$\Pr[\mathcal{A}(c) = 1 \mid b = 1] = 1$$

Substituting these values into the experiment formula:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] = \frac{1}{2}(1) + \frac{1}{2}(1) = 1$$

### 2.13
#### Question
Let $\Pi$ denote the Vigenère cipher where the message space consists of all 3-character strings (over the English alphabet), and the key is generated by first choosing the period $t$ uniformly from $\{1, 2, 3\}$ and then letting the key be a uniform string of length $t$.

(a) Define $\mathcal{A}$ as follows: $\mathcal{A}$ outputs $m_0 = \text{aab}$ and $m_1 = \text{abb}$. When given a ciphertext $c$, it outputs 0 if the first character of $c$ is the same as the second character of $c$, and outputs 1 otherwise. Compute $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right]$.

(b) Construct and analyze an adversary $\mathcal{A}'$ for which $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}', \Pi} = 1\right]$ is greater than your answer from part (a).

#### Solution
**(a) Calculation of $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right]$.** The choice of period $t$ happens uniformly at random from $\{1, 2, 3\}$, so $\Pr[t=1] = \Pr[t=2] = \Pr[t=3] = \frac{1}{3}$. Let $c = c[1]c[2]c[3]$ be the challenge ciphertext. The adversary $\mathcal{A}$ outputs 0 if $c[1] = c[2]$, and outputs 1 otherwise.

The success probability can be split across the three equally likely periods:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] = \sum_{i=1}^3 \frac{1}{3} \cdot \Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1 \;\middle|\; t=i\right]$$

Let's evaluate each period:

**1. When $t = 1$:** The key is a single letter $k_1$. Both $m_0[1]$ and $m_0[2]$ (or $m_1[1]$ and $m_1[2]$) are shifted by the same value $k_1$.
- If $b=0$ ($m_0 = \text{aab}$): $m_0[1]=\text{'a'}$, $m_0[2]=\text{'a'}$. Thus $c[1] = c[2] = (0+k_1)\bmod 26$, so $c[1]=c[2]$ is always true. $\mathcal{A}$ outputs $0$ (correct).
- If $b=1$ ($m_1 = \text{abb}$): $m_1[1]=\text{'a'}$, $m_1[2]=\text{'b'}$. Thus $c[1] = k_1$ and $c[2] = k_1+1$, so $c[1]=c[2]$ is never true. $\mathcal{A}$ outputs $1$ (correct).
- Therefore, $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1 \;\middle|\; t=1\right] = 1$.

**2. When $t = 2$:** The key is $(k_1, k_2)$. Characters $c[1]$ and $c[2]$ are shifted by completely independent uniform values $k_1$ and $k_2$.
- If $b=0$ ($m_0 = \text{aab}$): $c[1]=k_1$, $c[2]=k_2$. $\mathcal{A}$ outputs $0$ if $k_1 = k_2$, which occurs with probability $\frac{1}{26}$.
- If $b=1$ ($m_1 = \text{abb}$): $c[1]=k_1$, $c[2]=k_2+1$. $\mathcal{A}$ outputs $1$ if $k_1 \neq k_2+1$, which occurs with probability $\frac{25}{26}$.
- Therefore, $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1 \;\middle|\; t=2\right] = \frac{1}{2}\left(\frac{1}{26}\right) + \frac{1}{2}\left(\frac{25}{26}\right) = \frac{1}{2}$.

**3. When $t = 3$:** The key is $(k_1, k_2, k_3)$. Just like $t=2$, $c[1]$ and $c[2]$ are shifted by completely independent uniform values $k_1$ and $k_2$.
- By the same independent logic as $t=2$, $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1 \;\middle|\; t=3\right] = \frac{1}{2}$.

Combining the cases:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] = \frac{1}{3}(1) + \frac{1}{3}\left(\frac{1}{2}\right) + \frac{1}{3}\left(\frac{1}{2}\right) = \frac{2}{3} \approx 66.67\%$$

**(b) Construction and Analysis of a Stronger Adversary $\mathcal{A}'$.** To beat $\frac{2}{3}$, we want an adversary $\mathcal{A}'$ that can successfully identify *both* $t=1$ and $t=2$. We exploit the repeating key structure across the 1st and 3rd characters when $t \le 2$.

**Construction of $\mathcal{A}'$:**
1. $\mathcal{A}'$ outputs $m_0 = \text{aaa}$ and $m_1 = \text{aab}$.
2. Given a ciphertext $c$, it outputs 0 if $c[1] = c[3]$, and outputs 1 otherwise.

**Analysis of $\mathcal{A}'$:**

**1. When $t = 1$:** The key is $k_1$. $c[1]$ and $c[3]$ use the same shift $k_1$.
- If $b=0$ ($m_0 = \text{aaa}$): $m_0[1]=m_0[3]=\text{'a'}$, so $c[1]=c[3]$. $\mathcal{A}'$ outputs 0 (correct).
- If $b=1$ ($m_1 = \text{aab}$): $m_1[1]=\text{'a'}$, $m_1[3]=\text{'b'}$, so $c[3]=c[1]+1 \neq c[1]$. $\mathcal{A}'$ outputs 1 (correct).
- $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}', \Pi} = 1 \;\middle|\; t=1\right] = 1$.

**2. When $t = 2$:** The key sequence repeats every 2 characters, meaning $c[1]$ and $c[3]$ are **both shifted by $k_1$**.
- If $b=0$ ($m_0 = \text{aaa}$): $m_0[1]=m_0[3]=\text{'a'}$, so $c[1]=c[3]=k_1$. $\mathcal{A}'$ outputs 0 (correct).
- If $b=1$ ($m_1 = \text{aab}$): $m_1[1]=\text{'a'}$, $m_1[3]=\text{'b'}$, so $c[3]=k_1+1 \neq c[1]$. $\mathcal{A}'$ outputs 1 (correct).
- $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}', \Pi} = 1 \;\middle|\; t=2\right] = 1$.

**3. When $t = 3$:** The key is $(k_1, k_2, k_3)$. $c[1]$ is shifted by $k_1$ and $c[3]$ is shifted by $k_3$, which are independent.
- Due to the independent uniform shifts, $\mathcal{A}'$ wins with purely random guessing probability: $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}', \Pi} = 1 \;\middle|\; t=3\right] = \frac{1}{2}$.

Combining the cases for $\mathcal{A}'$:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}', \Pi} = 1\right] = \frac{1}{3}(1) + \frac{1}{3}(1) + \frac{1}{3}\left(\frac{1}{2}\right) = \frac{5}{6} \approx 83.33\%$$

Since $\frac{5}{6} > \frac{2}{3}$, $\mathcal{A}'$ is a strictly superior adversary.

### 2.14
#### Question
In this exercise, we look at different conditions under which the shift, mono-alphabetic substitution, and Vigenère ciphers are perfectly secret:
(a) Prove that if only a single character is encrypted, then the shift cipher is perfectly secret.
(b) What is the largest message space $\mathcal{M}$ for which the mono-alphabetic substitution cipher provides perfect secrecy?
(c) Prove that the Vigenère cipher using (fixed) period $t$ is perfectly secret when used to encrypt messages of length $t$.
Reconcile this with the attacks shown in the previous chapter.

#### Solution
**(a) Proof for the Shift Cipher (Single Character).** Let $\mathcal{M} = \{0, 1, \dots, 25\}$ be the message space representing a single English alphabet character, and let $\mathcal{K} = \{0, 1, \dots, 25\}$ be the key space where $k \in \mathcal{K}$ is chosen uniformly at random ($\Pr[K = k] = \frac{1}{26}$). The encryption algorithm is $\text{Enc}_k(m) = [m + k \bmod 26]$.

For any message $m \in \mathcal{M}$ and any ciphertext $c \in \{0, 1, \dots, 25\}$, we compute the probability that $m$ encrypts to $c$:
$$\Pr[\text{Enc}_K(m) = c] = \Pr[[m + K \bmod 26] = c] = \Pr[K = [c - m \bmod 26]]$$

Since $c - m \bmod 26$ always yields a unique value between $0$ and $25$, and $K$ is selected uniformly from this exact set:
$$\Pr[K = [c - m \bmod 26]] = \frac{1}{26}$$

Because $\Pr[\text{Enc}_K(m) = c] = \frac{1}{26}$ is a constant value independent of the choice of $m$, the scheme satisfies the definition of perfect secrecy.

---

**(b) Largest Message Space for the Mono-alphabetic Substitution Cipher.** The largest message space $\mathcal{M}$ consists of **all 1-character strings** (over the English alphabet, so $|\mathcal{M}| = 26$).

**Justification:**
- **Why it works for 1 character:** When encrypting only a single character, the mono-alphabetic substitution cipher maps that character to a randomly chosen unique substitute character out of 26 possibilities. This makes it equivalent to a shift cipher with a uniformly distributed random mapping, rendering the ciphertext distribution perfectly uniform and independent of the plaintext.
- **Why it fails for $\ge 2$ characters:** As soon as the message length is 2 or more, repeating characters in the plaintext create repeating characters in the ciphertext (e.g., the message $\text{"AA"}$ always yields a ciphertext of the form $cc$, whereas $\text{"AB"}$ never yields a ciphertext of the form $cc$). This structural pattern leaks plaintext information, destroying perfect secrecy.

---

**(c) Proof for the Vigenère Cipher (Length $t$) & Reconciliation.**

**Proof:** Let the message space $\mathcal{M} = \{0, 1, \dots, 25\}^t$ consist of all strings of length $t$. The key $K = (k_1, k_2, \dots, k_t)$ is chosen uniformly at random from $\mathcal{K} = \{0, 1, \dots, 25\}^t$, meaning each key component $k_i$ is chosen independently and uniformly with probability $\frac{1}{26}$.

For a message $m = m[1]\dots m[t]$ and ciphertext $c = c[1]\dots c[t]$, each character position $i$ (where $1 \le i \le t$) is encrypted independently:
$$c[i] = [m[i] + k_i \bmod 26]$$

For any fixed $m$ and $c$, the system of equations $c[i] \equiv m[i] + k_i \pmod{26}$ yields exactly one unique key sequence $K$:
$$k_i = [c[i] - m[i] \bmod 26] \quad \text{for } i = 1, \dots, t$$

Since the key components are chosen independently and uniformly at random:
$$\Pr[\text{Enc}_K(m) = c] = \prod_{i=1}^t \Pr[K_i = [c[i] - m[i] \bmod 26]] = \left(\frac{1}{26}\right)^t$$

Because $\Pr[\text{Enc}_K(m) = c] = 26^{-t}$ is totally independent of $m$, the scheme is perfectly secret.

**Reconciliation:** The attacks from the previous chapter succeed because they exploit the fact that in practice, messages are longer than the key period $t$, causing the key to repeat. When the message length equals $t$, each key character is used exactly once (equivalent to a one-time pad), so the ciphertext reveals no information.

### 2.15
#### Question
Give a direct proof that a scheme satisfying Definition 2.6 must have $|\mathcal{K}| \ge |\mathcal{M}|$. Specifically, let $\Pi$ be an arbitrary encryption scheme with $|\mathcal{K}| < |\mathcal{M}|$. Show an $\mathcal{A}$ for which $\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] > \frac{1}{2}$.

*(Note: The textbook contains a slight typographical error writing $|\mathcal{K}| \le |\mathcal{M}|$, but the core of Shannon's theorem dictates proving strict advantage when $|\mathcal{K}| < |\mathcal{M}|$.)*

#### Solution
**Proof.** Let $\Pi = (\text{Gen}, \text{Enc}, \text{Dec})$ be an encryption scheme where $|\mathcal{K}| < |\mathcal{M}|$. Following the hint, we construct a **randomized adversary $\mathcal{A}$** that exploits this gap to achieve a guessing advantage strictly greater than $1/2$.

**Construction of Adversary $\mathcal{A}$:**

1. $\mathcal{A}$ chooses two distinct messages $m_0, m_1 \in \mathcal{M}$ uniformly at random (so $m_0 \neq m_1$).
2. $\mathcal{A}$ outputs $m_0, m_1$ and receives the challenge ciphertext $c \leftarrow \text{Enc}_K(m_b)$ where $b \in \{0, 1\}$ is a fair coin flip.
3. Let $V(c) = \{\text{Dec}_k(c) \mid k \in \mathcal{K}\}$ be the set of all possible messages that could decrypt from $c$. Notice that because each key yields at most one decryption, $|V(c)| \le |\mathcal{K}|$. Since $|\mathcal{K}| < |\mathcal{M}|$, it follows that $|V(c)| < |\mathcal{M}|$.
4. $\mathcal{A}$ makes its guess $b'$ according to the following rules:
   - If $m_0 \in V(c)$ and $m_1 \notin V(c)$, output $b' = 0$.
   - If $m_1 \in V(c)$ and $m_0 \notin V(c)$, output $b' = 1$.
   - Otherwise (if both are in $V(c)$ or neither is in $V(c)$), flip a fair coin and output the result.

**Analysis of Success Probability.** By the correctness of encryption, the true message $m_b$ used to create $c$ must reside in $V(c)$.

- **Case $b = 0$:** Here, $m_0 \in V(c)$ is guaranteed. Thus, the scenario ($m_1 \in V(c) \land m_0 \notin V(c)$) can never happen.
  - If $m_1 \notin V(c)$, $\mathcal{A}$ outputs $0$ (correct) with probability $1$.
  - If $m_1 \in V(c)$, $\mathcal{A}$ outputs $0$ (correct) with probability $1/2$.

  Therefore, the conditional success probability is:
  $$\Pr[\mathcal{A}(c) = 0 \mid b = 0] = 1 \cdot \Pr[m_1 \notin V(c) \mid b=0] + \frac{1}{2} \cdot \Pr[m_1 \in V(c) \mid b=0]$$
  $$\phantom{\Pr[\mathcal{A}(c) = 0 \mid b = 0]} = 1 - \frac{1}{2}\Pr[m_1 \in V(c) \mid b=0]$$

  Since $m_1$ was chosen uniformly at random from the remaining $|\mathcal{M}| - 1$ elements, and the remaining space in $V(c)$ contains at most $|V(c)| - 1 \le |\mathcal{K}| - 1$ elements:
  $$\Pr[m_1 \in V(c) \mid b=0] \le \frac{|\mathcal{K}| - 1}{|\mathcal{M}| - 1}$$
  $$\Pr[\mathcal{A}(c) = 0 \mid b = 0] \ge 1 - \frac{|\mathcal{K}| - 1}{2(|\mathcal{M}| - 1)}$$

- **Case $b = 1$:** By perfect symmetry, $m_1 \in V(c)$ is guaranteed, leading to the same bound:
  $$\Pr[\mathcal{A}(c) = 1 \mid b = 1] \ge 1 - \frac{|\mathcal{K}| - 1}{2(|\mathcal{M}| - 1)}$$

**Total Success Probability:**
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] = \frac{1}{2}\Pr[\mathcal{A}(c) = 0 \mid b = 0] + \frac{1}{2}\Pr[\mathcal{A}(c) = 1 \mid b = 1] \ge 1 - \frac{|\mathcal{K}| - 1}{2(|\mathcal{M}| - 1)}$$

Because we started with the premise that $|\mathcal{K}| < |\mathcal{M}|$, we know that $|\mathcal{K}| - 1 < |\mathcal{M}| - 1$. This makes the fraction strictly less than 1:

$$\frac{|\mathcal{K}| - 1}{|\mathcal{M}| - 1} < 1 \implies \frac{|\mathcal{K}| - 1}{2(|\mathcal{M}| - 1)} < \frac{1}{2}$$

Thus:
$$\Pr\left[\text{PrivK}^{\text{eav}}_{\mathcal{A}, \Pi} = 1\right] > 1 - \frac{1}{2} = \frac{1}{2}$$

The adversary achieves a success probability strictly greater than $1/2$, violating perfect indistinguishability. Therefore, any perfectly secure scheme must have $|\mathcal{K}| \ge |\mathcal{M}|$.

The following questions concern multiple encryptions of single-character ASCII plaintexts with the one-time pad using the same 8-bit key. You may assume that the plaintexts are either (upper- or lower-case) English letters or the space character.
(a) Say you see the ciphertexts $1011\;0111$ and $1110\;0111$. What can you deduce about the plaintext characters these correspond to?

**Fundamental ASCII Properties.** Let the 8 bits of a character be indexed from left to right as $b_1, b_2, \dots, b_8$.

- **Space character (`0x20`):** `0010 0000` $\implies$ the second bit ($b_2$) is always **0**.
- **Uppercase letters (`0x41` - `0x5A`):** `0100 0001` to `0101 1010` $\implies$ the second bit ($b_2$) is always **1**.
- **Lowercase letters (`0x61` - `0x7A`):** `0110 0001` to `0111 1010` $\implies$ the second bit ($b_2$) is always **1**.

Thus, **all English letters have a second bit of 1**, while the **space character has a second bit of 0**. When encrypted with the same key bit $k_2$, all letters will share the same ciphertext second bit ($\neg k_2$), while a space character will have a flipped ciphertext second bit ($k_2$).

---

**(a) Analysis of $1011\;0111$ and $1110\;0111$.**
- $c^{(1)} = 1\mathbf{0}11\;0111 \implies$ second bit is **0**.
- $c^{(2)} = 1\mathbf{1}10\;0111 \implies$ second bit is **1**.

Since their second bits differ, one message must be a space character and the other must be an English letter. We compute the XOR difference between the two ciphertexts to eliminate the key:
$$m^{(1)} \oplus m^{(2)} = c^{(1)} \oplus c^{(2)} = 1011\;0111 \oplus 1110\;0111 = 0101\;0000 = \text{0x50}$$

Since one of them is a space (`0x20`), we find the other character by XORing it with `0x50`:
$$\text{0x20} \oplus \text{0x50} = \text{0x70} = \text{'p'}$$

**Deduction:** One of the plaintexts is the **space character**, and the other is the lowercase letter **'p'**. (We cannot determine which ciphertext corresponds to which character without knowing the key.)

---

**(b) Analysis of $0110\;0110$, $0011\;0010$, and $0010\;0011$.** Let's isolate the second bit of each ciphertext:
- $c^{(1)} = 0\mathbf{1}10\;0110 \implies$ second bit is **1**.
- $c^{(2)} = 0\mathbf{0}11\;0010 \implies$ second bit is **0**.
- $c^{(3)} = 0\mathbf{0}10\;0011 \implies$ second bit is **0**.

Since all English letters must share the same second ciphertext bit under the same key, $c^{(2)}$ and $c^{(3)}$ (which share the bit 0) must be letters. The outlier, $c^{(1)}$, must be the **space character**.

Knowing that $m^{(1)} = \text{space} = \text{0x20} = 0010\;0000$, we can uniquely recover the entire 8-bit key $k$:
$$k = c^{(1)} \oplus m^{(1)} = 0110\;0110 \oplus 0010\;0000 = 0100\;0110$$

Now, we can fully decrypt $m^{(2)}$ and $m^{(3)}$:
- $m^{(2)} = c^{(2)} \oplus k = 0011\;0010 \oplus 0100\;0110 = 0111\;0100 = \text{0x74} = \text{'t'}$
- $m^{(3)} = c^{(3)} \oplus k = 0010\;0011 \oplus 0100\;0110 = 0110\;0101 = \text{0x65} = \text{'e'}$

#### Question
Assume we require only that an encryption scheme $(\text{Gen}, \text{Enc}, \text{Dec})$ with message space $\mathcal{M}$ satisfy the following: For all $m \in \mathcal{M}$, we have $\Pr[\text{Dec}_K(\text{Enc}_K(m)) = m] \geq 2^{-t}$. (This probability is taken over choice of the key as well as any randomness used during encryption/decryption.)

Show that perfect secrecy can be achieved with $|\mathcal{K}| < |\mathcal{M}|$ when $t \geq 1$. Prove a lower bound on the size of $\mathcal{K}$ in terms of $t$.

#### Solution
**Part 1: Counterexample for $|\mathcal{K}| < |\mathcal{M}|$ when $t \geq 1$.** Let $t \geq 1$ and choose a message space such that $1 < |\mathcal{M}| \leq 2^t$.

Define a scheme with a single key $\mathcal{K} = \{k_0\}$, so $|\mathcal{K}| = 1 < |\mathcal{M}|$:
- **Enc$_{k_0}(m)$**: Always outputs a fixed ciphertext $c_0$ for all $m \in \mathcal{M}$.
- **Dec$_{k_0}(c_0)$**: Outputs a message $m' \in \mathcal{M}$ chosen uniformly at random.

1. **Perfect Secrecy**: Since $\Pr[C = c_0 \mid M = m] = 1$ for all $m$, the ciphertext leaks no information about the message. Perfect secrecy holds.
2. **Correctness Condition**: For any $m \in \mathcal{M}$, the decryption yields the correct message with probability:
   $$\Pr[\text{Dec}_K(\text{Enc}_K(m)) = m] = \frac{1}{|\mathcal{M}|}$$
   Since we chose $|\mathcal{M}| \leq 2^t$, it follows that $\frac{1}{|\mathcal{M}|} \geq 2^{-t}$, satisfying the required condition.

**Part 2: Lower Bound on $|\mathcal{K}|$.** From Part 1, when $|\mathcal{M}| \leq 2^t$, a valid scheme can be constructed using only a single key. Since any valid encryption scheme requires at least one key to exist, the absolute lower bound on the size of the key space is:
$$|\mathcal{K}| \geq 1$$

### 2.18
#### Question
Let $\epsilon > 0$ be a constant. Say an encryption scheme is $\epsilon$-*perfectly secret* if for every adversary $\mathcal{A}$ it holds that
$$\Pr[\text{PrivK}_{\mathcal{A},\Pi}^{\text{eav}} = 1] \leq \frac{1}{2} + \epsilon.$$
(Compare to Definition 2.6.) Consider a variant of the one-time pad where $\mathcal{M} = \{0, 1\}^\ell$ and the key is chosen uniformly from an arbitrary set $\mathcal{K} \subseteq \{0, 1\}^\ell$ with $|\mathcal{K}| = (1 - \epsilon) \cdot 2^\ell$; encryption and decryption are otherwise the same.

(a) Prove that this scheme is $\epsilon$-perfectly secret.
(b) Prove that this scheme is $\left(\frac{\epsilon}{2(1-\epsilon)}\right)$-perfectly secret when $\epsilon \leq 1/2$. (Note that $\left(\frac{\epsilon}{2(1-\epsilon)}\right) \leq \epsilon$ here, so this is an improvement over part (a).)
(c) Prove that any deterministic scheme that is $\epsilon$-perfectly secret must have $|\mathcal{K}| \geq (1 - 2\epsilon) \cdot |\mathcal{M}|$. (Note: It is an open question to prove a tight lower bound that also holds for randomized schemes.)

#### Solution
**Part (a).** Let $U$ denote the uniform distribution over the ciphertext space $\{0, 1\}^\ell$. For any message $m$, its ciphertext distribution $C_m$ is uniform over a specific subset of size $|\mathcal{K}| = (1 - \epsilon)2^\ell$.

The Statistical Distance (SD) between $C_m$ and $U$ is:
$$\text{SD}(C_m, U) = \frac{1}{2} \left( |\mathcal{K}| \left| \frac{1}{|\mathcal{K}|} - \frac{1}{2^\ell} \right| + (2^\ell - |\mathcal{K}|) \left| 0 - \frac{1}{2^\ell} \right| \right) = 1 - \frac{|\mathcal{K}|}{2^\ell} = \epsilon$$

By the triangle inequality, for any two messages $m_0, m_1$:
$$\text{SD}(C_{m_0}, C_{m_1}) \leq \text{SD}(C_{m_0}, U) + \text{SD}(U, C_{m_1}) = 2\epsilon$$

An adversary's maximum probability of winning the distinguishability game is bounded by $\frac{1}{2} + \frac{1}{2}\text{SD}(C_{m_0}, C_{m_1}) \leq \frac{1}{2} + \epsilon$. Thus, the scheme is $\epsilon$-perfectly secret.

**Part (b).** To find a tighter bound, we calculate the exact maximum distance. Let $S_0 = \{m_0 \oplus k \mid k \in \mathcal{K}\}$ and $S_1 = \{m_1 \oplus k \mid k \in \mathcal{K}\}$.

The exact SD between the two ciphertext distributions is given by the non-overlapping mass:
$$\text{SD}(C_{m_0}, C_{m_1}) = \frac{|S_0 \setminus S_1|}{|\mathcal{K}|}$$

Because both $S_0$ and $S_1$ are subsets of $\{0, 1\}^\ell$, their union cannot exceed the total space size $2^\ell$. Therefore:
$$|S_0 \setminus S_1| = |S_0 \cup S_1| - |S_1| \leq 2^\ell - |\mathcal{K}| = 2^\ell - (1-\epsilon)2^\ell = \epsilon 2^\ell$$

Substitute this into the SD formula:
$$\text{SD}(C_{m_0}, C_{m_1}) \leq \frac{\epsilon 2^\ell}{(1-\epsilon)2^\ell} = \frac{\epsilon}{1-\epsilon}$$

The maximum success probability is $\frac{1}{2} + \frac{1}{2}\text{SD}(C_{m_0}, C_{m_1}) \leq \frac{1}{2} + \frac{\epsilon}{2(1-\epsilon)}$.

**Part (c).** In a deterministic scheme, let $M(c) = \{m \in \mathcal{M} \mid \exists k \in \mathcal{K}, \text{Enc}_k(m) = c\}$. Because each key maps a given message to exactly one ciphertext, there are at most $|\mathcal{K}|$ messages that can result in the same ciphertext $c$. Thus, $|M(c)| \leq |\mathcal{K}|$.

Fix an arbitrary message $m_0$. Choose another message $m_1$ uniformly at random from $\mathcal{M}$. We evaluate the expected statistical distance:
$$\mathbb{E}_{m_1}[\text{SD}(C_{m_0}, C_{m_1})] \geq 1 - \sum_{c \in \text{supp}(C_{m_0})} \Pr[C_{m_0}=c] \cdot \Pr_{m_1}[c \in \text{supp}(C_{m_1})]$$

Since $\Pr_{m_1}[c \in \text{supp}(C_{m_1})] = \frac{|M(c)|}{|\mathcal{M}|} \leq \frac{|\mathcal{K}|}{|\mathcal{M}|}$:
$$\mathbb{E}_{m_1}[\text{SD}(C_{m_0}, C_{m_1})] \geq 1 - \sum_{c} \Pr[C_{m_0}=c] \frac{|\mathcal{K}|}{|\mathcal{M}|} = 1 - \frac{|\mathcal{K}|}{|\mathcal{M}|}$$

Because the expected value is at least $1 - \frac{|\mathcal{K}|}{|\mathcal{M}|}$, there must exist a specific pair $(m_0, m_1)$ such that $\text{SD}(C_{m_0}, C_{m_1}) \geq 1 - \frac{|\mathcal{K}|}{|\mathcal{M}|}$.

For the scheme to be $\epsilon$-perfectly secret, the advantage must be $\leq \epsilon$, meaning:
$$\frac{1}{2}\text{SD}(C_{m_0}, C_{m_1}) \leq \epsilon \implies \frac{1}{2}\left(1 - \frac{|\mathcal{K}|}{|\mathcal{M}|}\right) \leq \epsilon$$

Solving for $|\mathcal{K}|$ yields:
$$|\mathcal{K}| \geq (1 - 2\epsilon) |\mathcal{M}|$$

### 2.19
#### Question
In this problem we consider definitions of perfect secrecy for the encryption of *two* messages (using the same key). Here we consider distributions on *pairs* of messages from the message space $\mathcal{M}$; we let $M_1, M_2$ be random variables denoting the first and second message, respectively. (These random variables are not assumed to be independent.) We generate a (single) key $k$, sample a pair of messages $(m_1, m_2)$ according to the given distribution, and then compute ciphertexts $c_1 \leftarrow \text{Enc}_k(m_1)$ and $c_2 \leftarrow \text{Enc}_k(m_2)$; this induces a distribution on pairs of ciphertexts and we let $C_1, C_2$ be the corresponding random variables.

**(a)** Say encryption scheme $(\text{Gen}, \text{Enc}, \text{Dec})$ is *perfectly secret for two messages* if for all distributions on $\mathcal{M} \times \mathcal{M}$, all $m_1, m_2 \in \mathcal{M}$, and all ciphertexts $c_1, c_2 \in \mathcal{C}$ with $\Pr[C_1 = c_1 \land C_2 = c_2] > 0$:
$$\Pr[M_1 = m_1 \land M_2 = m_2 \mid C_1 = c_1 \land C_2 = c_2] = \Pr[M_1 = m_1 \land M_2 = m_2].$$
Prove that *no* encryption scheme can satisfy this definition.
**Hint:** Take $c_1 = c_2$.

**(b)** Say encryption scheme $(\text{Gen}, \text{Enc}, \text{Dec})$ is *perfectly secret for two distinct messages* if for all distributions on $\mathcal{M} \times \mathcal{M}$ where the first and second messages are guaranteed to be different (i.e., distributions on pairs of *distinct* messages), all $m_1, m_2 \in \mathcal{M}$, and all $c_1, c_2 \in \mathcal{C}$ with $\Pr[C_1 = c_1 \land C_2 = c_2] > 0$:
$$\Pr[M_1 = m_1 \land M_2 = m_2 \mid C_1 = c_1 \land C_2 = c_2] = \Pr[M_1 = m_1 \land M_2 = m_2].$$
Show an encryption scheme that provably satisfies this definition.
**Hint:** The encryption scheme you propose need not be efficient, although an efficient solution is possible.

#### Solution
**Part (a).** Assume such a scheme exists. Consider a prior message distribution where for two distinct messages $m_1 \neq m_2$, the probability $\Pr[M_1 = m_1 \land M_2 = m_2] > 0$.

Let the generated ciphertexts be identical, meaning $c_1 = c_2 = c$. Because decryption is deterministic for a fixed key ($\text{Dec}_k(c)$ must yield a single unique message), identical ciphertexts strictly imply that the original messages were identical ($M_1 = M_2$).

Therefore, the posterior probability of them being distinct given identical ciphertexts is zero:
$$\Pr[M_1 = m_1 \land M_2 = m_2 \mid C_1 = c \land C_2 = c] = 0$$

This contradicts the definition which requires the posterior probability to equal the strictly positive prior probability. Thus, no such scheme can exist.

**Part (b).** Let the message space be $\mathcal{M}$ and the ciphertext space be $\mathcal{C} = \mathcal{M}$. Let the key space $\mathcal{K}$ consist of all possible permutations of $\mathcal{M}$, chosen uniformly at random.

- **Enc**: $c = \pi_k(m)$
- **Dec**: $m = \pi_k^{-1}(c)$

For any two distinct messages $m_1, m_2$ and any two distinct ciphertexts $c_1, c_2$, the number of permutations that map $m_1 \to c_1$ and $m_2 \to c_2$ is $(|\mathcal{M}| - 2)!$. The total number of permutations is $|\mathcal{M}|!$.

The probability of generating these specific ciphertexts given the messages is:
$$\Pr[C_1 = c_1 \land C_2 = c_2 \mid M_1 = m_1 \land M_2 = m_2] = \frac{(|\mathcal{M}| - 2)!}{|\mathcal{M}|!} = \frac{1}{|\mathcal{M}|(|\mathcal{M}| - 1)}$$

Since this conditional probability is a constant value independent of the specific choices of the distinct messages $m_1, m_2$, applying Bayes' theorem shows that the posterior probability is exactly equal to the prior probability. This scheme provably satisfies the definition.

*(Note: An efficient alternative is evaluating an affine function $c = a\cdot m + b \pmod p$ over a finite field where the key is $(a,b)$ and $a \neq 0$.)*
